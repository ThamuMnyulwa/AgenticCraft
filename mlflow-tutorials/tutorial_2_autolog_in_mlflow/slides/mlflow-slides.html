03
Why it matters

The problem it solves

  • Lost experiments. Which hyperparameters gave that good score last Tuesday? MLflow remembers so you don't have to.
  • Manual bookkeeping. No more copy-pasting metrics into spreadsheets after every run.
  • Reproducibility gaps. Every run captures its code version, parameters, and environment.
  • Framework sprawl. One consistent interface across scikit-learn, PyTorch, XGBoost, and more.
04
Autologging

Activating autologging

Autologging captures parameters, metrics, and models automatically, without adding logging calls to your training code.

import mlflow

# One call, before you train
mlflow.autolog()

# Your normal training code, unchanged
model.fit(X_train, y_train)
✦

mlflow.autolog() enables logging for every supported library you've imported. Framework-specific variants exist too, like mlflow.sklearn.autolog().

05
Under the hood

What autolog captures

Parameters

Model hyperparameters and fit arguments, pulled directly from the estimator.

Metrics

Training scores, loss curves, and evaluation results per epoch or step.

Artifacts

The serialized model, plus plots and summaries where the flavour supports them.

Metadata

Run timing, source code version, and the model signature for later serving.

06
The big idea

Zero-config coding

Instrumentation without ceremony. You write ML code the way you always have; MLflow observes and records it for you.

  • No manual log calls. Skip log_param and log_metric for every value. Autolog infers them.
  • No boilerplate runs. A run starts and ends around your fit automatically.
  • Add detail only when you want it. Drop into manual logging for custom metrics, and autolog fills in the rest.
07
The difference

Manual vs zero-config

Before 
with mlflow.start_run():
    mlflow.log_param("n_est", 100)
    mlflow.log_param("depth", 6)
    model.fit(X, y)
    acc = model.score(X, y)
    mlflow.log_metric("acc", acc)
    mlflow.sklearn.log_model(
        model, "model")
After 
mlflow.autolog()

model.fit(X, y)

# params, metrics, and
# the model: all logged
08
Example 1  ·  scikit-learn

True zero-config

scikit-learn works with generic autolog out of the box. One call, then train as normal.

import mlflow
from sklearn.ensemble import RandomForestClassifier
from sklearn.datasets import load_iris
from sklearn.model_selection import train_test_split

mlflow.autolog()   # <- the only MLflow line you add

X, y = load_iris(return_X_y=True)
X_tr, X_te, y_tr, y_te = train_test_split(X, y, test_size=0.2)

clf = RandomForestClassifier(n_estimators=100, max_depth=6)
clf.fit(X_tr, y_tr)   # params, metrics, model, signature: all logged

Example 2  ·  PyTorch

Autolog via Lightning

Vanilla PyTorch uses a custom training loop, so MLflow can't autolog it directly. PyTorch Lightning gives autolog the structure it needs.

import mlflow
import lightning as L

mlflow.autolog()   # detects Lightning; logs metrics, params, checkpoints

class LitModel(L.LightningModule):
    def training_step(self, batch, idx):
        x, y = batch
        loss = self.loss_fn(self(x), y)
        self.log("train_loss", loss)   # surfaced by autolog
        return loss

trainer = L.Trainer(max_epochs=5)
trainer.fit(model, train_loader)   # run tracked automatically
⚠

For a raw PyTorch loop, use manual mlflow.log_metric() calls. Autolog is a Lightning feature.

12
One platform, two modes

MLflow for ML vs GenAI

Everything so far is the classic ML path. MLflow 3 added a parallel track for LLMs and agents, where the unit of work is a prompt or a trace, not a fit() call.

Traditional ML

Train & track

Unit: a training run.
Log: mlflow.autolog() captures params, metrics, model.
Evaluate: mlflow.models.evaluate() for classification and regression.
Compare: runs side by side in the UI.

GenAI & agents

Trace & judge

Unit: a prompt or a trace.
Log: auto-tracing captures prompts, tool calls, tokens, cost.
Evaluate: mlflow.genai.evaluate() with LLM-as-judge scorers.
Manage: versioned Prompt Registry.

✦

Same registry, same UI, same lineage. If you run both training pipelines and LLM apps, they live in one place.
